What is CLIPS?

- CLIPS is a forwardŚchaining, pattern-matching
knowledge-based system shell.

- CLIPS stands for C Language /mplementation Production
System.

- CLIPS was developed by the Artificial Intelligence Section
of the Jonnson Space Center of NASA.

- Its pattern-matching algorithm is based on the Rete
algorithm.

- CLIPS provides high portability, low cost, and high degree
of integration with external programs.

Facts and Rules

- CLIPS operates by maintaining a list of facts and a set of
rules which operate on them.

- A fact is a piece of information such as (colour green) or
(parent_of John Susan).

- Facts are created by asserting them onto the fact
database using the assert command.

CLIPS> (assert (colour green) )
<Fact-1>

The <Fact-1> part is the response from CLIPS to say that a
new fact (fact number 1) has been placed on the fact
database.
Facts and Rules

> The (facts) command will list all current facts.

CLIPS> (facts)

f-O (initial-fact)

f-1 (colour green)
For a total of 2 facts.

- Facts may also be retracted (removed) from the fact
database by using the retract command.

CLIPS> (retract 1)
CLIPS> (facts)

f-O (initial-fact)

For a total of 1 fact.
Facts and Rules

- Two things to note here:

1. firstly, to retract a fact you must specify a number (the
fact-index), not the fact itself, and

2. secondly, fact-indices are not reused. Once fact O has
been retracted, the next fact asserted will have the
index 2, not O.
Facts and Rules

- Arule is expressed in the form ĹIF something is true THEN do
some action.
(defrule duck
(animal-is duck)
=>
(assert (Sound-is quack))).

- The rule consists of three parts.

1. (defrule duck, simply gives the rule a unique name.

2. (animal-is duck), is the pattern (the IF part) of the rule and
3. (assert (sound-is quack)), is the action (the THEN part).

> In plain language, this rule means Ĺif there is a fact (animal-is
duck) on the fact database, then assert another fact, (soundis quack), onto the fact databaseĺ.

Templates

- Some facts that we created are link to each other.

- For example, the following facts are related to each other:
(age Johan 21)
(weight Johan 80)
(height Johan 175)
(blood-pressure Johan 130 80)

- The above facts can be grouped by using the deftemplate
function.
Template

(deftemplate personal-data
(slot name)
(slot age)
(slot weight)
(slot height)
(multislot blood-pressure) )

- deftemplate does not any facts, but rather the form which
facts can take.

- Every time a fact of this type is created, it contains the slots
specified in its definition, each of which can contain a value
and can be accessed by name.

(assert (personal-data (name Johan) (age 21)

Template

- Template facts can be altered without the need to retract them and
assert a new version.

> The function modify allows you to change the value of one or more
slots on a fact.

- Suppose it's Johan's birthday. So we will asserting the fact
(birthday Johan). If we define the rule

(defrule birthday
birthday <- (birthday ?name)
?data-fact <- (personal-data (name ?name) (age ?age))

(modify ?data-fact (age (+ ?age 1))) (retract ?birthday) )

> The rule will modify Johan's age while leaving all his other personal
data intact.

Truth maintenance

- And the rule below

(defrule cardiac-risk

(personal-data (name ?name) (Smoker yes) (weight ?weight))
(test (> 2weight 100))

(assert (cardiac-risk ?name)) )

- Tests whether a person is overweight and a smoker.
- If he is, then it asserts a warning that his heart is at risk.
- What happens if the person gives up smoking, or loses weight?

> The warning fact will still be present, because it is not linked in any
way to the rule which created it.

- This means that the fact base is no longer consistent with reality.
OO
Truth maintenance

- We can overcome the problem by using the logical element.
- The previous rule can be modified to:

(defrule cardiac-risk
(logical (personal-data (name ?name) (Smoker yes)
(weight ?weight)))
(logical (test (> ôweight 100)))

(assert (cardiac-risk ?name)) )

- When the above rule is run, the results are identical to those
generated by the first version in slide 15 (i.e. the facts
(cardiac-risk brenda) and (cardiac-risk charles) are asserted).
OO
Truth maintenance

- The difference occurs when we change the initial data.

- Make sure the fact window is open, then having run the rule,
locate the fact-index of Brenda's personal data (Suppose on
your system, it's 2) and type at the command line:

(modify 2 (weight 80))

- Now the fact (cardiac-risk brenda) disappears from the fact
list.

- By using the logical keyword, we have created a link between
the fact asserted and the premises on which it was based.

- When the premises changed, their results were no longer
valid, so they were removed.
Salience

> One way to control rule firing.

- Asalience value is between -10000 and 10000, where 0 is
the default value.

- Rules with a higher salience are fired before rules with a
lower salience.
Salience

- Consider the following two rules:
(defrule poke-fun-at-smokers

(personal-data (name ?name) (smoker yes))
=>

(printout t ?name " is a fool." crlf) )

(defrule worry-about-thin-people
(personal-data (name ?name) (weight ?weight))
(test (< ?weight 80))

(printout t ?name " is looking a bit thin." crlf) )
When you run those rules, then they will both fire, but the order
of their firing will depend only upon the order in which the facts
on which they depend were created.
Salience

- When we modify the previous two rules into:
(defrule poke-fun-at-smokers

(declare (salience 10))

(personal-data (name ?name) (smoker yes))

(printout t ?name " is a fool.ö crlf) )
(defrule worry-about-thin-people
(declare (salience 20))
(personal-data (name ?name) (weight ?weight))

(test (< ?weight 80))

(printout t ?name " is looking a bit thin." crlf) )

Then the higher salience of rule worry-about-thin-people will ensure
that it fires first.

Deffunction

- User-defined function

- Comprised of five elements:
1) a name,
2) an optional comment,
3) a list of zero or more required parameters,
4) an optional wildcard parameter to handle a variable number of
arguments and

5) a sequence of actions, or expressions, which will be executed in
order when the deffunction is called.
Deffunction

(deffunction <name> [<comment>] (<regular-parameter>* [<wildcard-parameter>])
<action>*)

<regular-parameter> ::= <single-field-variable>
<wildcard-parameter> ::= <multifield-variable>

(deffunction print-args (?a ?b $?c)
(printout t ?a "" 2b" and " (length ?c) " extras: " ?c crlf))

Function call & Output

CLIPS> (print-args 1 2)
1 2 and 0 extras: ()

CLIPS> (print-args a b c d)
ab and 2 extras: (c d)
